# Documentation

This directory keeps lightweight project notes that should not be generated from CLI output.

- `decisions/`: accepted architecture and governance decisions.
- `design/`: behavior and user-experience notes for notifications and plugin setup.
- `tech/`: implementation references for hook processing, settings, and plugin packaging.
- `release/`: release, publishing, and rollback notes.
- `postmortems/`: incident, regression, and release follow-up reports.
